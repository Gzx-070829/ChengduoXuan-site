# 项目结构与改进建议（面向初学者）

## 1) 当前技术栈

这个项目是一个 **Next.js 站点**，可以理解为“用 React 做页面、用 Next.js 管路由和构建”的现代前端项目。

- **Next.js 16（App Router）**：负责页面路由、构建、启动服务。
- **React 19**：负责组件化页面开发。
- **TypeScript 5**：给代码加类型，减少低级错误。
- **Tailwind CSS 4（已安装）**：用于样式开发，但当前主要还是用内联 style。
- **ESLint 9 + eslint-config-next**：用于代码规范检查。

---

## 2) 主要文件和文件夹是做什么的

> 下面按“新手最常看”的顺序整理：

### 根目录

- `package.json`
  - 项目依赖和脚本入口（`npm run dev/build/start/lint`）。
- `tsconfig.json`
  - TypeScript 编译配置（例如是否严格类型检查）。
- `next.config.ts`
  - Next.js 配置文件，目前几乎是默认空配置。
- `eslint.config.mjs`
  - ESLint 的规则配置。
- `postcss.config.mjs`
  - CSS 处理配置（配合 Tailwind）。
- `README.md`
  - 项目说明文档，目前是 create-next-app 默认模板。

### `app/`（核心业务目录）

- `app/layout.tsx`
  - 全局布局。所有页面都会套在这个布局里。
  - 这里引入了全局样式和顶部导航栏 `Navbar`。
- `app/page.tsx`
  - 首页（`/`）。
  - 现在主要展示个人站介绍和栏目卡片。
- `app/globals.css`
  - 全局 CSS，含主题色变量与基础字体设置。
- `app/components/Navbar.tsx`
  - 导航栏组件。
- `app/components/Card.tsx`
  - 首页卡片组件（目前用 JS 事件做 hover 动画，所以写成了客户端组件）。
- `app/ai/page.tsx`
  - AI 栏目列表页（`/ai`）。
- `app/ai/[slug]/page.tsx`
  - AI 文章详情动态路由页（`/ai/xxx`），目前还是占位内容。

### `public/`

- 放静态资源（svg 图标等），可直接通过路径访问。

## 3) 最值得优先改进的 3 个地方

下面是“投入小、收益高、适合当前阶段”的优先级：

### 优先级 1：补齐缺失页面，避免导航 404

现在导航里有 `/bci`、`/programming`、`/literature`、`/daily`、`/projects`，但这些路由页面还没有。

- **问题**：用户点了会报 404，体验会明显下降。
- **建议**：先建最简页面（标题 + 简介 + “建设中”），保证都能打开。
- **收益**：马上提升站点完整度。

### 优先级 2：统一样式方案（逐步从内联 style 迁移）

目前样式大多写在组件内部（内联 style），而且 Tailwind 已经装好了但没充分使用。

- **问题**：后续页面一多，样式很难复用，改视觉会比较痛苦。
- **建议**：先从 `Navbar` 和 `Card` 开始，迁移到 `globals.css` / Tailwind class。
- **收益**：代码更整洁，后续改版更轻松。

### 优先级 3：让内容系统可扩展（文章数据与详情页打通）

现在 AI 栏目的文章列表是写死数组，详情页是占位文本。

- **问题**：新增文章要改代码，不利于长期写作。
- **建议**：先做“本地数据文件驱动”（例如 `data/articles.ts`），再逐步升级到 MDX 或 CMS。
- **收益**：后续写内容更高效，网站更像真正博客。

## 一个清晰、可执行的小步修改计划（先不大改）

### 第 1 步（半天）

1. 新建 5 个缺失栏目页面（BCI / Programming / Literature / Daily / Projects）。
2. 每页放统一模板：标题、简介、返回首页链接。
3. 手动点一遍导航，确认无 404。

### 第 2 步（0.5~1 天）

1. 抽离重复样式（按钮、卡片、页面容器）。
2. `Card` 的 hover 动画改为 CSS（减少 JS 事件代码）。
3. 保持视觉不变，只改实现方式。

### 第 3 步（1 天）

1. 新建文章数据文件（标题、slug、摘要、正文占位）。
2. `/ai` 页面读取数据渲染列表。
3. `/ai/[slug]` 根据 slug 渲染对应文章；找不到时返回 404。

### 第 4 步（可选，后续迭代）

1. 更新 `README.md`：补项目定位、目录结构、开发流程。
2. 增加基础 SEO（更好的 `metadata` 标题和描述）。
3. 再考虑是否引入 MDX、标签系统、搜索等功能。
